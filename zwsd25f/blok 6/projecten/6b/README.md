# Projectweek 6B - CRUD verdieping op 6A

In 6B bouw je verder op jouw bestaande 6A-project. Je start dus niet opnieuw.

## Doel

Pas de kennis uit hoofdstuk 1 t/m 5 toe in een realistische uitbreiding:
- Resource routes en controllers
- CRUD formulieren met validatie
- Relaties en eager loading
- Auth, middleware, rollen en gates
- Policies voor acties op individuele records

## Algemene eisen voor alle 6B-projecten

- Werk verder in je bestaande 6A-repository.
- Gebruik je nog geen login/registratie? Volg dan eerst Hoofdstuk 4 - Authenticatie en Middleware (Laravel Breeze) uit de lessen in blok 6.
- Gebruik ten minste 1 resource controller voor een kernentiteit.
- Implementeer volledige CRUD voor minimaal 2 entiteiten.
- Voeg server-side validatie toe bij store en update.
- Gebruik flash messages bij create, update en delete.
- Beveilig schrijf-acties met auth middleware.
- Pas rolcontrole toe (bijvoorbeeld admin/manager/medewerker/lid).
- Gebruik minimaal 1 gate en minimaal 1 policy.
- Toon relationele gegevens op detailpagina's met klikbare links.
- Gebruik centrale layout, minimaal 2 Blade componenten en centrale CSS.
- Zorg dat seeddata alle 6B user stories testbaar maakt.

## Oplevering

1. Werkende applicatie met seeddata.
2. Korte demo flow per rol.
3. Korte toelichting in README van je project:
   - Welke CRUD's zijn gemaakt
   - Welke middleware/gate/policy zijn gebruikt
   - Welke validatieregels zijn toegepast
